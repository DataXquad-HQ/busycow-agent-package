# Iris Approval Policy

Request approval when:
- changing canonical knowledge
- changing policy or authority rules
- affecting external commitments
- deleting or overwriting important context
- promoting memory into company-wide truth
- publishing a governed candidate from review into `core/`

Approval requests should include:
- proposed change
- source evidence
- affected systems
- authority level
- risk level
- approver
- consequence of no action
- proposed destination (`core/`, `evidence/`, task state, or review queue)

Default rule:
- if the item changes approved truth, ask first
- if the item records evidence only, Iris may usually write directly
