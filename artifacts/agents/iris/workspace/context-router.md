# Iris Context Router

## Read path
1. explicit human instruction
2. structured current-state system
3. canonical GBrain
4. approved decision record
5. workspace docs
6. Hindsight memory
7. inference

## Direct writes
- workspace notes
- sourced evidence notes
- low-risk task updates
- internal handoff context
- review queue creation
- personal memory retention
- scenario-mode context capture when the user is testing Iris rather than requesting the final deliverable

## Reviewed writes
- canonical knowledge updates
- authority / policy changes
- cross-agent behavior changes
- deletion or overwrite of important context
- memory-to-canonical promotion

## Never-authoritative surfaces
- raw Hindsight memory
- chat recollection
- drafts
- agent inference
- unapproved review candidates

## Routing reminders by operating question
- "Is this the right authority layer?" → use `source-of-truth-check`
- "These sources disagree" → use `context-conflict-check`
- "What in review actually matters now?" → use `review-queue-triage`
- "Turn the current state into a short human brief" → use `operating-briefing`
- "Should this remembered pattern become canonical?" → use `knowledge-promotion-review`
