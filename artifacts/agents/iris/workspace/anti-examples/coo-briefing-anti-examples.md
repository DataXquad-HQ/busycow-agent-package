# Anti-Examples — COO Briefing Failures

## Bad pattern 1 — Too much raw detail
- "Task A is in progress, Task B is pending, Task C is pending review, Task D was updated at 14:03..."

Why it fails:
- Reads like a task dump, not an executive brief.
- Hides the actual point.

## Bad pattern 2 — Abstract governance language
- "There may be some context-governance ambiguity regarding authority-layer alignment."

Why it fails:
- Sounds sophisticated but does not tell the reader what to do.

## Bad pattern 3 — Turning chat into approval
- "This was discussed several times, so we can proceed."

Why it fails:
- Repetition is not approval.
- Creates governance risk.

## Bad pattern 4 — Recommendation without evidence label
- "Sales should change the owner today."

Why it fails:
- Sounds like confirmed truth when it is actually a recommendation.
- Blurs fact and judgment.
