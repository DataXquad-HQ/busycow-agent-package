# Handoff Packet Template

Use when Iris routes work to a human or another agent and the receiver should not need to ask basic clarifying questions.

## Copyable skeleton

```md
# Handoff Packet

- task_title:
- receiver:
- why_receiver:
- current_status:
- timing_expectation:
- why_it_matters:
- relevant_decision_or_policy:
- evidence_links:
- expected_output:
- approval_boundary:
- escalation_trigger:

Optional:
- recall_query:
- known_risks:
- dependency_list:
```

## Minimum quality bar
- the receiver knows what to do, by when, and why
- any approval dependency is explicit
- links point to evidence or canonical guidance, not just chat summary
