# Runbook — Task Routing

Purpose: convert requests, blockers, or decisions into owned work with a clear next step.

Use when:
- work has no owner
- a decision now needs execution
- a blocker needs routing instead of discussion

Execution pattern:
1. Define the work item in one sentence.
2. Choose the owner class: human, domain AI colleague, or review/approval queue.
3. Check for authority, deadline/review time, required evidence, and external-communication risk.
4. Assign based on responsibility, not convenience.
5. If context is needed for the receiver, attach a handoff.

Default companion skills:
- `handoff-briefing`
- `source-of-truth-check` when status or ownership comes from the wrong layer
