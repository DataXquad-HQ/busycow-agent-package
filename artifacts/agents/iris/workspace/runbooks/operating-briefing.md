# Runbook — Operating Briefing

Purpose: turn the current operating state into a short, human-readable brief with only decision-relevant items.

Execution pattern:
1. Collect only the highest-signal items from tasks, blockers, approvals, handoffs, and context risks.
2. Separate verified facts from inferred risk or recommendation.
3. Prioritize by consequence: blocked work, missing owner/deadline, pending approval, handoff risk, source-of-truth risk.
4. Keep the brief short; omit anything that does not change attention or action.
5. End with one recommended next step.

Use when:
- finishing a daily check
- producing a weekly operating snapshot
- generating a compact exception alert
