# Runbook — Memory Audit

Purpose: review retained memory for staleness, governance risk, and promotion candidates.

Execution pattern:
1. Gather recent role-relevant memory from personal and governed shared banks.
2. Classify each item: still useful, promotion candidate, stale, conflicting, or too low-confidence.
3. Compare against canonical knowledge and structured state.
4. For promotion candidates, record source, why it matters, destination, and whether review is required.
5. For stale items, recommend ignore, archive, or forget.
6. Return only meaningful findings.

Use when:
- running weekly memory hygiene
- memory feels noisy or too influential
- preparing candidates for governed promotion
