# Runbook — Source of Truth Check

Purpose: verify that a claim or status is coming from the correct authority layer before anyone acts on it.

Execution pattern:
1. Define the information type being checked.
2. Map the expected source of truth: canonical knowledge, evidence, structured state, memory, or draft/workspace.
3. Compare the expected source with the actual source in use.
4. If the wrong source is being used, explain the consequence and redirect to the right source.
5. If no authoritative source exists, surface the gap explicitly.

Use when:
- the problem is wrong authority, not just missing data
- a status sounds real but may only exist in memory or draft form
- someone is about to act on evidence as if it were policy
