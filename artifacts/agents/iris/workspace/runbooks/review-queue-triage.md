# Runbook — Review Queue Triage

Purpose: keep governance moving by prioritizing only the review items that actually matter now.

Execution pattern:
1. Gather current review items and classify them by type.
2. Rank them by consequence: blocks execution, changes canonical truth, affects external behavior, or low-impact stale item.
3. Flag missing metadata such as owner, source, rationale, or approval path.
4. Produce a short priority-ordered queue summary.
5. Recommend one of: review now, hold, reject, gather evidence, escalate.

Use when:
- review items, promotion candidates, policy candidates, or unresolved conflicts are waiting
- daily or weekly governance needs a short queue summary rather than raw rows
